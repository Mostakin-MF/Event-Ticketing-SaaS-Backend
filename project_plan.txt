1) Vision / Goal
- Build a multi-tenant event ticketing SaaS platform with Next.js (frontend) and NestJS (backend), using PostgreSQL with TypeORM. Prioritize clean architecture, DX, performance (SSR/ISR), and a clear separation between UI and API layers.
- Focus on event organizers creating public event pages, selling tickets, processing payments, and handling attendee check-in via QR codes.

2) High-Level Requirements
- Multi-tenant architecture with data isolation per organizer
- User auth (JWT in HttpOnly cookies) with role-based access (owner/admin/staff)
- Public event pages (SSR/ISR) and authenticated organizer portal (CSR)
- Event management, ticket types, inventory management
- Secure checkout and payment processing
- QR code generation and mobile check-in system
- Sales dashboard and reporting with CSV export
- Email notifications for tickets and confirmations

3) Scope (In / Out)
- In: Next.js app, NestJS REST API, PostgreSQL, TypeORM, Docker Compose for local DB, lint/format, basic CI, payment integration (Stripe), email service
- Out: Advanced microservices, real-time websockets (for now), cloud infra IaC (initially), advanced features like reserved seating (V2)

4) Tech Stack (Current Implementation)
- Frontend: Next.js 16.0.0 (TypeScript, App Router), Tailwind CSS 4.0, React 19.2.0, Axios for data fetching
- Backend: NestJS 11.0.1 (TypeScript), REST API, class-validator/transformer, TypeORM (to be added)
- Database: PostgreSQL (to be configured)
- DevOps/Hosting: Docker Compose (local Postgres+Adminer), Vercel (web) + Render/Railway/Fly (API) [TBD]
